## 前提

- 開発: Claude Code
- LLM実行: Claude Code CLI（`claude -p`）／Cowork＋ローカルrunnerサーバー方式。従量課金のAnthropic APIキーは使用せず、既存のClaudeサブスクリプションの範囲で実行する
- 対象: ツールA（店舗マスタ作成）／ツールA'（店舗情報自動反映ツール）／ツールB（差分検知）／Slack連携（起動・通知・差分反映承認）

---

## 見積金額

| ツール | 内容 | 金額（税抜） |
|---|---|---:|
| **ツールA** | HPBを第一情報源とした店舗情報取得、公式HPフォールバック補完、Googleスプレッドシートへの店舗マスタ構築 | 120,000円 |

| **Slack連携（オプション）** | スラッシュコマンドでの各ツール起動、実行結果・差分検知サマリの通知、差分検知通知の「反映する」ボタンによる承認→ツールA'自動起動 | 30,000円 |

## 備考

- 上記はフルスコープ（ツールA・A'・B・Slack連携すべて）での金額です。Slack連携は運用の利便性向上のためのオプション機能であり、CLIからの直接実行でも全機能が問題なく利用できます
- 公式HPフォールバック（HPBに情報がない項目を店舗公式HPから補完する機能）はツールAの金額に含みます
- 差分検知後の「Slack承認→ツールA'が差分店舗のみ自動反映」の連携機能は、ツールA'・Slack連携双方の金額に含みます（追加費用なし）。WordPress更新を伴うため完全自動化はせず、担当者のSlack上での承認を必須とする設計です
- 開発はClaude Codeによる実装を前提とし、Claude Code CLI（サブスクリプション内）でのLLM実行によりAPI従量課金は発生しません
- 大量処理（店舗一括取得・400記事一括移行）はClaudeサブスクリプションの使用枠（約5時間ごとにリセット）を考慮し、複数セッションに分割して実行します（詳細は[設計書.md](./設計書.md)参照）
- 段階的な発注も可能です。ツールAのみを先行発注し、効果検証後にツールA'・B・Slack連携を追加発注する形にも対応します
