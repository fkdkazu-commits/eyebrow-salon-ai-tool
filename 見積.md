## 前提

- 開発: Claude Code
- LLM実行: Claude Code CLI（`claude -p`）／Cowork＋ローカルrunnerサーバー方式。従量課金のAnthropic APIキーは使用せず、既存のClaudeサブスクリプションの範囲で実行する
- 対象: ツールA（店舗マスタ作成）／ツールA'（店舗情報自動反映ツール）／ツールB（差分検知・承認済み差分の自動反映）
- 差分検知後の記事反映は、変更履歴シートの`approved`列（チェックボックス）による承認方式とする（Slack連携は採用しない）

## 備考

- 公式HPフォールバック（HPBに情報がない項目を店舗公式HPから補完する機能）はツールAの金額に含みます
- 差分検知後の「承認→ツールA'が差分店舗のみ自動反映」の連携機能は、ツールA'・ツールB双方の金額に含みます（追加費用なし）。WordPress更新を伴うため完全自動化はせず、担当者が変更履歴シート上で承認（approved列をTRUEに変更）することを必須とする設計です
- 大量処理（店舗一括取得・400記事一括移行）はClaudeサブスクリプションの使用枠（約5時間ごとにリセット）を考慮し、複数セッションに分割して実行します（詳細は[設計書.md](./設計書.md)参照）
- 各ツールは記載の納品予定日を目安に順次納品します（ツールA → ツールA' → ツールB）
